# Copilot Usage Guide

## Good use cases
- DTOs and schema mapping boilerplate.
- Mapper/converter helpers.
- Unit and integration test generation.
- Repetitive adapter glue code.
- Small utility/helper functions.

## Bad use cases
- Making architecture decisions without ADR alignment.
- Large uncontrolled rewrites.
- Inventing product rules not in source-of-truth docs.
- Bypassing tenancy/permission/audit requirements for speed.

## Workflow
- Start from explicit task template.
- Constrain editable files.
- Require test and standards checklist in output.
